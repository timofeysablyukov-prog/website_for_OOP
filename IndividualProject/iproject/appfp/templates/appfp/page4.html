Инкапсуляция

Что такое инкапсуляция?

Инкапсуляция - это механизм объединения данных (полей) и методов (функций), которые работают с этими данными, в одной единице - классе. Также инкапсуляция включает контроль доступа к данным.

Уровни доступа в Python

Python использует соглашения об именовании для определения уровней доступа:

class Example:
    def __init__(self):
        self.public = "Доступно отовсюду"      # публичный атрибут
        self._protected = "Доступно в классе и наследниках"  # защищённый
        self.__private = "Доступно только в классе"  # приватный

Как вы наверно поняли, перед именем защизённого атрибута ствится одно "_", а перед приватным две

Пример инкапсуляции

class Temperature:
    def __init__(self, celsius):
        self.__celsius = celsius  # приватное поле
    def get_celsius(self):
        return self.__celsius
    def set_celsius(self, value):
        if -273.15 <= value <= 1000:  # проверка корректности
            self.__celsius = value
        else:
            print("Некорректная температура")
    def get_fahrenheit(self):
        return self.__celsius * 9/5 + 32

Здесь представлен класс температуры, где мы можем задавать её в цельсиях и выводить в тех же цельсиях или фаренгейтах (метод get_farenheit).

В этом примере поле __celsius инкапсулировано - доступ к нему возможен только через методы get_celsius() и set_celsius(). Это позволяет контролировать корректность данных.

Зачем нужна инкапсуляция?

1. Защита данных от некорректного изменения

2. Сокрытие внутренней реализации

3. Гибкость в изменении реализации

4. Упрощение интерфейса класса
